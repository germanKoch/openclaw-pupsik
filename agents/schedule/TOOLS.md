/# Schedule Agent — Справочник

## TickTick — Структура проектов

Актуальный список проектов: `get_user_projects()`

Типовая структура (обновляй при изменениях):

| Проект | Назначение |
|--------|-----------|
| DAILY/PERIODIC | Ежедневные рутины и повторяющиеся задачи |
| ALTERACTS | Рабочие задачи |
| CLAWD | Заметки агента (финансовые отчёты, дневник) |
| Inbox | Входящие (без проекта) |

### Приоритеты задач

| Значение | Приоритет |
|----------|-----------|
| 0 | Нет |
| 1 | Низкий |
| 3 | Средний |
| 5 | Высокий |

### Формат дат

```
dueDate: "2026-03-04T09:00:00+03:00"  # ISO 8601 с таймзоной
```

## Теги-нотация

TickTick поддерживает вложенные теги через `/`. Три типа структурированных тегов для хранения метаинформации о задачах:

### Формат тегов

| Тип | Родительский тег | Дочерний формат | Пример | Семантика |
|-----|-----------------|-----------------|--------|-----------|
| Дедлайн | `DUE` | `D_YYYY.MM.DD` | `DUE/D_2026.12.31` | Реальный дедлайн (отдельно от scheduling date) |
| Earliest start | `NOT_EARLIER` | `N_YYYY.MM.DD` | `NOT_EARLIER/N_2026.06.01` | Задача неактуальна до этой даты |
| Человек | `PERSON` | `P_ИМЯ` | `PERSON/P_KSENIIA` | Задача связана с человеком |

**Формат даты**: `YYYY.MM.DD` с точками (не тире).

### Ключевое разделение DUE vs dueDate

```
dueDate задачи  = дата ВЫПОЛНЕНИЯ (когда запланировано)   → например: вторник
DUE/D_* тег    = РЕАЛЬНЫЙ ДЕДЛАЙН (когда должно быть готово) → например: пятница
```

### Конструкция тегов

```
DUE:          "DUE/D_" + YYYY + "." + MM + "." + DD
NOT_EARLIER:  "NOT_EARLIER/N_" + YYYY + "." + MM + "." + DD
PERSON:       "PERSON/P_" + ИМЯ_ЛАТИНИЦЕЙ_ВЕРХНИЙ_РЕГИСТР
              пробел в имени → нижнее подчёркивание
              "Ivan Petrov" → "PERSON/P_IVAN_PETROV"
              "Ксения" → "PERSON/P_KSENIIA"
```

### Отображение тегов (человекочитаемый вид)

| Сырой тег | Отображение |
|-----------|-------------|
| `DUE/D_2026.04.30` | `дедлайн: 30 апр 2026` |
| `NOT_EARLIER/N_2026.06.01` | `заморожена до: 1 июн 2026` |
| `PERSON/P_KSENIIA` | `связана с: Kseniia` |
| `PERSON/P_IVAN_PETROV` | `связана с: Ivan Petrov` |

**Никогда не показывать сырые теги пользователю.**

### Горящий дедлайн

Задача с `DUE/*` тегом считается «горящей» если: `дедлайн ≤ сегодня + 7 дней`.
- `days_until = 0` → «дедлайн СЕГОДНЯ»
- `days_until < 0` → «дедлайн ПРОСРОЧЕН N дн.»
- `0 < days_until ≤ 7` → горящий (показать через N дней)

### Обработка граничных случаев

| Ситуация | Действие |
|----------|----------|
| Несколько `DUE/*` тегов | Использовать ближайший, предупредить: «Найдено несколько дедлайнов, используется ближайший» |
| `NOT_EARLIER > DUE` (начало после дедлайна) | Предупредить при создании/обнаружении |
| Невалидная дата (`D_2026.13.01`) | Игнорировать, сообщить: «Невалидный тег [тег], пропущен» |
| `NOT_EARLIER` будущее + `DUE` в течение 7 дней | Показать в горящих дедлайнах с пометкой ⚠️ «заморожена до [date]» |
