# Планирование дня

Собрать план на сегодня: добавить рутины, подтянуть рабочие задачи, порекомендовать задачи на основе целей и застоявшихся проектов — согласовав всё с пользователем.

## Контекстные файлы

Перед началом загрузи с диска (если существуют):

- **Предпочтения и цели:** `~/.openclaw/workspace/daily-planner/preferences.md`
- **Статистика:** `~/.openclaw/workspace/daily-planner/stats.md`
- **Знание о проектах:** `~/.openclaw/workspace/ticktick-inbox/projects.md` (общий с ticktick-inbox)

Эти файлы обновляются по итогам каждого запуска.

---

## Алгоритм

### Фаза 0. Собрать контекст

1. Прочитать `preferences.md`, `stats.md`, `projects.md` если существуют.
2. Определить день недели и является ли сегодня рабочим днём (пн–пт = рабочий, если в preferences нет другого графика).
3. Получить данные из TickTick:
   - Список проектов: `ticktick.get_projects`
   - Задачи на сегодня (с dueDate = today): собрать из всех проектов
   - Задачи из проекта DAILY/PERIODIC
   - Если рабочий день — задачи из проекта ALTERACTS
   - Задачи из всех остальных проектов (для анализа застоявшихся)

---

### Фаза 1. Автоматический блок — рутины и обязательные

Эти задачи добавляются в день **без согласования** (просто информируем пользователя):

**1a. Рутины (DAILY/PERIODIC):**
- Все задачи из проекта DAILY/PERIODIC автоматически включаются в план дня.
- Если задача повторяющаяся и уже есть экземпляр на сегодня — не дублировать.

**1b. Уже запланированные:**
- Задачи из любых проектов с dueDate = сегодня автоматически включаются.

**1c. Рабочие задачи (только рабочие дни):**
- Все задачи из проекта ALTERACTS автоматически включаются в план.
- Если в ALTERACTS есть задачи с приоритетом High — вынести их первыми.

Вывод:
```
📅 План на {дата}, {день недели}

🔁 Рутины (DAILY/PERIODIC): N задач
- [ ] Задача 1
- [ ] Задача 2

📌 Уже запланировано на сегодня: M задач
- [ ] Задача (проект: X)

💼 Работа (ALTERACTS): K задач
- [ ] ⚡ Высокий приоритет: Задача
- [ ] Задача
```

---

### Фаза 2. Рекомендации — на основе целей и застоя

Это блок **согласования**: бот предлагает, пользователь выбирает.

**2a. Задачи, продвигающие цели:**
- Загрузить цели из `preferences.md`
- Найти задачи во всех проектах, которые явно продвигают текущие цели
- Выбрать 2-4 наиболее импактные, которых ещё нет в плане

**2b. Застоявшиеся задачи и проекты:**
- Найти задачи, не обновлявшиеся >14 дней
- Найти проекты, где последняя активность >14 дней назад
- Выбрать 1-3 задачи из таких проектов, предпочитая задачи с приоритетом

**2c. Просроченные задачи:**
- Задачи с dueDate < сегодня
- Предложить: перенести на сегодня / перепланировать / убрать дедлайн

Вывод:
```
🎯 Рекомендую добавить (продвигают цели):
1. «Задача» (проект: X) — цель: «Запустить Y»
2. «Задача» (проект: Z) — цель: «Выучить W»

🧊 Застоявшееся (давно без движения):
3. «Задача» (проект: X, не трогали 23 дня)
4. Проект «Y» — 0 активности 18 дней, задач: 12

⏰ Просроченное:
5. «Задача» (дедлайн был 3 дня назад)

Какие добавить в план? (номера через запятую, или «все», или «пропустить»)
```

**Ждать ответа пользователя.** Добавить выбранные задачи в план.

---

### Фаза 3. Контроль нагрузки

После формирования плана — сверить с исторической статистикой из `stats.md`.

- Посчитать общее количество задач в плане.
- Сравнить со средним количеством закрытых задач в день (из stats.md).

Правила:
- Если задач в плане **>150% от среднего** — предупредить:
  > ⚠️ В плане {N} задач, а ты обычно закрываешь ~{avg}. Может убрать что-то?
- Если задач **<50% от среднего** — подсказать:
  > 💡 В плане всего {N} задач (обычно ~{avg}). Хочешь добавить что-то ещё?
- Если статистики ещё нет (первый запуск) — не предупреждать, просто собирать данные.

**Если предупреждение сработало — ждать ответа пользователя.** Дать возможность скорректировать план.

---

### Фаза 4. Финальный план

После всех согласований вывести итоговый план:

```
✅ Финальный план на {дата}  ({N} задач, обычно ~{avg})

🔁 Рутины:
- [ ] ...

💼 Работа:
- [ ] ...

🎯 Цели:
- [ ] ...

📋 Остальное:
- [ ] ...

Хорошего дня! 🚀
```

### Фаза 5. Обновить файлы

**stats.md** — обновить статистику:

Формат:
```
# Статистика планирования
_Обновлено: {дата}_

## Средние показатели
- Среднее задач запланировано в день: {число}
- Среднее задач закрыто в день: {число}
- Дней с данными: {число}

## Последние 14 дней
| Дата | Запланировано | Закрыто |
|------|--------------|---------|
| ... | ... | ... |
```

Логика обновления:
- При планировании — записать количество запланированных задач на сегодня.
- При следующем запуске — посмотреть сколько задач из вчерашнего плана были закрыты (completedDate = вчера) и обновить колонку «Закрыто».
- Пересчитать средние.

**preferences.md** — обновить если пользователь сообщил новую информацию:

Формат:
```
# Предпочтения и цели
_Обновлено: {дата}_

## График
- Рабочие дни: пн-пт
- Выходные: сб-вс

## Цели
- {формулировка цели} — приоритет: {высокий/средний}
- ...

## Предпочтения по планированию
- {любые заметки: «не люблю больше 3 рабочих задач в день», «утро — для глубокой работы» и т.п.}

## Заметки бота
- {выводы из наблюдений: «обычно закрывает рабочие задачи первыми», «в пятницу планирует меньше» и т.п.}
```

Сохранить оба файла в `~/.openclaw/workspace/daily-planner/`.

---

### Ограничения

- НЕ создавать и не удалять задачи автоматически. Только рекомендовать.
- НЕ менять приоритеты и дедлайны без согласования.
- Все действия с данными пользователя — через согласование (кроме рутин из Фазы 1).
- Если `preferences.md` не существует — при первом запуске спросить пользователя о его целях и графике.
