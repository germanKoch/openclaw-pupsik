# Финансовый анализ

Сформировать детальный финансовый отчёт за месяц: доходы, расходы по категориям, сравнение с предыдущим месяцем, тренды, аномалии и рекомендации. Данные берутся из ZenMoney.

## Хранение данных

История финансового анализа хранится в TickTick, чтобы Хозяин мог видеть результаты в любое время.

- **Проект:** `CLAWD` (project_id: `698c924c8f08f21724f5daba`)
- **Заметка:** `financial_analysis` — нужно найти task_id по названию при первом запуске
- Если заметка `financial_analysis` не существует — создать задачу с таким названием в проекте CLAWD.

Чтение: `ticktick.get_task_by_ids projectId=698c924c8f08f21724f5daba taskId=<task_id>`
Запись: `ticktick.update_task` с `taskId`, `id`, `projectId`, `content=<обновлённый контент>`

## Алгоритм

### Фаза 0. Загрузить контекст

1. Найти заметку `financial_analysis` в проекте CLAWD:
   - Вызвать `ticktick.get_project_with_data projectId=698c924c8f08f21724f5daba`
   - Найти задачу с названием `financial_analysis`
   - Если не найдена — создать: `ticktick.create_task` с `title=financial_analysis`, `projectId=698c924c8f08f21724f5daba`, `content=# Financial Analysis History\n`
   - Запомнить `task_id`
2. Прочитать содержимое заметки (`content` поле). Файл содержит JSON-блоки с итогами предыдущих месяцев.
3. Определить период отчёта:
   - Если запуск 1-го числа месяца (автоматический крон) → **предыдущий календарный месяц** целиком.
   - Если запуск в любой другой день (ручной вызов) → **текущий месяц с 1-го числа по вчера** включительно. В отчёте пометить: «⏳ Неполный месяц (данные по {дата})».
4. Запомнить `date_from` и `date_to` для дальнейших запросов.

---

### Фаза 1. Получить данные из ZenMoney

1. Вызвать `zenmoney.get_accounts` — получить список счетов с балансами и валютами. Составить маппинг account_id → валюта.
2. Вызвать `zenmoney.get_categories` — получить список категорий.
3. Вызвать `zenmoney.get_transactions` с параметрами:
   - `date_from` = первый день периода (YYYY-MM-DD)
   - `date_to` = последний день периода (YYYY-MM-DD)
   - `limit` = 500
4. Вызвать `zenmoney.get_budgets` с теми же `date_from` / `date_to`.
5. Для каждой транзакции определить валюту по account_id из маппинга счетов.
6. Разделить транзакции на три группы по признаку суммы и контекста:
   - **Расходы**: сумма отрицательная, не перевод между своими счетами
   - **Доходы**: сумма положительная, не перевод между своими счетами
   - **Переводы**: транзакции между собственными счетами (исключить из доходов/расходов)

---

### Фаза 2. Агрегация и анализ

**Для каждой валюты отдельно:**

1. **Итоги:**
   - Общий доход = сумма всех доходных транзакций
   - Общие расходы = сумма всех расходных транзакций (абсолютное значение)
   - Переводы = сумма всех переводов (абсолютное значение)
   - Баланс = доход − расходы

2. **Расходы по категориям:**
   - Сгруппировать расходы по категории
   - Для каждой категории: сумма, доля от общих расходов (%)
   - Транзакции без категории → группа «Без категории»
   - Отсортировать по сумме убывания

3. **Топ-5 категорий расходов:** первые 5 из отсортированного списка.

4. **Сравнение с предыдущим месяцем (MoM):**
   - Если в `history.md` есть данные за предыдущий месяц:
     - Для дохода, расходов, баланса: абсолютное изменение и процент
     - Для каждой категории расходов: процент изменения
     - Направление: ↑ если выросло >5%, ↓ если упало >5%, → если ±5%
   - Если нет данных за предыдущий месяц: пропустить сравнение, написать «нет данных за предыдущий месяц».

5. **Аномалии:**
   - Для каждой категории вычислить среднюю сумму одной транзакции за месяц.
   - Найти транзакции, превышающие 2× среднюю для своей категории.
   - Для каждой аномалии запомнить: дату, сумму, получателя (payee), категорию.

---

### Фаза 3. Тренды и кросс-корреляции

**Выполнять только если в `history.md` есть данные за 3+ месяцев.**

Если данных меньше 3 месяцев — полностью пропустить эту фазу.

1. **Тренды по категориям:**
   - Для каждой из топ-10 категорий расходов (по текущему месяцу):
     - Собрать суммы за последние 3–6 месяцев из history.md
     - Определить тренд: ↑ Рост (если категория выросла >5% два месяца подряд), ↓ Снижение (если упала >5% два месяца подряд), → Стабильно (иначе)
     - Записать суммы: X → Y → Z руб.

2. **Кросс-корреляции:**
   - Для каждой пары категорий из топ-10 определить направление изменения за каждый месяц (рост/падение/стабильно).
   - Если две категории двигались в одном направлении 3+ раз — отметить как положительную корреляцию.
   - Если двигались в противоположных направлениях 3+ раз — отметить как отрицательную.
   - Ограничиться 3 наиболее выраженными наблюдениями.
   - Сформулировать на русском: «{Категория A} и {Категория B} растут вместе последние N месяцев».

### Фаза 4. Оценка и рекомендации

Это **главная аналитическая часть** отчёта. Здесь ты выступаешь как личный финансовый аналитик Хозяина. Будь конкретным, честным и полезным.

#### 4.1. Оценка накоплений

**Обязательная секция.** Определи, идёт ли Хозяин вперёд по накоплениям или тратит больше, чем зарабатывает.

- Вычислить **норму сбережений** = (доход − расходы) / доход × 100%
- Если есть история за предыдущие месяцы — показать динамику нормы сбережений: X% → Y% → Z%
- Дать чёткую оценку:
  - Норма >20%: «Отличный месяц для накоплений. Ты откладываешь {X}% дохода — это выше рекомендуемых 20%.»
  - Норма 10–20%: «Неплохо — {X}% дохода ушло в накопления. Это {сумма} руб.»
  - Норма 0–10%: «Впритык — удалось сохранить только {X}% дохода ({сумма} руб.). Основные статьи, съевшие бюджет: {топ-3 категории}.»
  - Норма <0%: «Этот месяц в минусе: расходы превысили доходы на {сумма} руб. Главные причины: {топ-3 категории с наибольшим расходом}.»
- Если есть данные за 2+ месяцев — посчитать **среднюю норму сбережений** и сравнить текущий месяц со средней.

#### 4.2. Общая оценка месяца

**Обязательная секция.** Дать развёрнутую, честную оценку финансового месяца. Писать от второго лица на русском, тон — дружелюбный, но прямой (как хороший финансовый советник).

Рассмотреть:
- Как текущий месяц соотносится с предыдущими (лучше/хуже/так же)?
- Какие категории «выстрелили» неожиданно?
- Есть ли позитивные тенденции, которые стоит отметить?
- Есть ли тревожные сигналы?
- Пример: «Февраль был крепким месяцем. Доходы выросли на 15%, а расходы удалось удержать. Единственное — подписки незаметно подросли до {X} руб., и это уже третий месяц подряд. Продукты тоже выше среднего — возможно, сезонный эффект. В целом, ты укрепил финансовую подушку на {Y} руб.»

Если это первый месяц анализа — написать базовую оценку без сравнений, но с конкретными наблюдениями.

#### 4.3. Детальные рекомендации

Сгенерировать **5–8 конкретных** рекомендаций. Каждая ОБЯЗАНА ссылаться на конкретные категории, суммы и даты.

Типы рекомендаций (применять все подходящие):

1. **Перерасход:** категория выросла >20% к прошлому месяцу → «Расходы на {категорию} выросли на {X}% ({сумма} руб. vs {сумма прошлого месяца} руб.). Стоит проверить — это разовое или новая норма?»
2. **Аномалия:** аномальная транзакция → «Крупная трата: {payee} на {сумма} руб. ({дата}) — это в {N}× больше среднего по {категория}. Если это плановая покупка — ок, если нет — обрати внимание.»
3. **Позитивный тренд:** расходы снизились → «Расходы снизились на {X}% к прошлому месяцу. Основной вклад: {категория} (−{сумма} руб.). Хороший результат!»
4. **Бюджет:** категория превысила бюджет → «{Категория}: использовано {X}% бюджета ({факт} руб. из {бюджет} руб.).»
5. **Растущий тренд:** категория растёт 3+ месяца → «{Категория} растёт уже {N} месяцев: {суммы}. Стоит пересмотреть — это осознанный рост или ползучий?»
6. **Структура расходов:** если одна категория >30% всех расходов → «{Категория} составляет {X}% всех расходов. Это {сумма} руб. — стоит подумать, можно ли оптимизировать.»
7. **Подписки:** если категория подписок есть → всегда упомянуть: «Подписки: {сумма} руб./мес. Проверь, всё ли используется. Даже {минимальная подписка} руб. — это {сумма×12} руб. в год.»
8. **Накопления:** если норма сбережений падает 2+ месяца → «Норма сбережений падает: {X}% → {Y}% → {Z}%. Если тренд продолжится, через N месяцев ты будешь тратить больше, чем зарабатывать.»

Если данных для глубоких рекомендаций мало (первый запуск) — всё равно дать 3–5 наблюдений на основе текущих данных. Не писать «данных недостаточно» — **всегда** есть что сказать по текущему месяцу.

НЕ давать общие советы типа «экономьте больше» — только конкретика с цифрами.

### Фаза 5. Сформировать отчёт

Собрать отчёт в формате Markdown, пригодном для Telegram:

```
📊 Финансовый отчёт за {месяц на русском} {год}

💰 Итоги

| Показатель | Сумма | Изменение |
|------------|-------|-----------|
| Доходы     | X руб. | +Y% ↑   |
| Расходы    | X руб. | -Y% ↓   |
| Переводы   | X руб. |          |
| Баланс     | X руб. | +Y% ↑   |

📋 Расходы по категориям

| Категория | Сумма | Доля | vs прошлый месяц |
|-----------|-------|------|-------------------|
| {cat}     | X руб. | Y% | +Z% ↑            |
| ...       |        |     |                   |

🔍 Аномалии

- ⚠️ {дата}: {payee} — {сумма} руб. ({категория}), в {N}× больше среднего

📈 Тренды

| Категория | Тренд | Последние N месяцев |
|-----------|-------|---------------------|
| {cat}     | ↑ Рост | X → Y → Z руб.    |

🔗 Кросс-корреляции

- {cat1} и {cat2} растут вместе последние N месяцев

📊 Бюджет vs Факт

| Категория | Бюджет | Факт | Использовано |
|-----------|--------|------|--------------|
| {cat}     | X руб. | Y руб. | Z% ⚠️    |

💰 Накопления

Норма сбережений: X% (доход Y руб. − расходы Z руб. = +W руб.)
Динамика: A% → B% → X%
{Оценка: отличный/хороший/слабый/отрицательный месяц для накоплений}

📝 Оценка месяца

{2–4 предложения: развёрнутая, честная оценка финансового месяца от лица аналитика — сравнение с прошлым, что выросло/упало, общий вердикт}

💡 Рекомендации

1. {конкретная рекомендация с цифрами}
2. {конкретная рекомендация с цифрами}
3. {конкретная рекомендация с цифрами}
...
```

**Правила формирования:**

- «Итоги», «Расходы по категориям», «Накопления», «Оценка месяца», «Рекомендации» — **всегда** присутствуют.
- «Аномалии» — только если найдены аномальные транзакции.
- «Тренды» и «Кросс-корреляции» — только при 3+ месяцах истории.
- «Бюджет vs Факт» — только если `get_budgets` вернул данные. ⚠️ ставить при использовании >90%.
- Секции без данных (кроме обязательных) **полностью убираются**.
- Если есть транзакции в нескольких валютах — для каждой валюты отдельные секции «Итоги» и «Расходы по категориям» с пометкой валюты. Накопления считать по основной валюте (RUB).
- Если месяц неполный (ручной запуск) — в заголовке добавить: «⏳ Неполный месяц (данные по {дата})».

### Фаза 6. Сохранить историю

Записать сводку текущего месяца в заметку `financial_analysis` в TickTick (проект CLAWD).

1. Прочитать текущий `content` заметки (task_id из Фазы 0).
2. Сформировать JSON-блок для текущего месяца.
3. Если блок за текущий период уже есть в content — **заменить** его.
4. Если нет — **вставить** новый блок после заголовка (чтобы последние данные были первыми).
5. Обновить заметку: `ticktick.update_task` с `taskId`, `id`, `projectId=698c924c8f08f21724f5daba`, `content=<обновлённый контент>`

Формат content заметки:

```
# Financial Analysis History

## {YYYY-MM}

```json
{
  "period": "{YYYY-MM}",
  "generated_at": "{ISO 8601 с часовым поясом}",
  "currencies": {
    "RUB": {
      "total_income": 150000.0,
      "total_expenses": 120000.0,
      "total_transfers": 30000.0,
      "net": 30000.0,
      "savings_rate": 20.0,
      "categories": {
        "Health & Fitness": -15000.0,
        "Transport": -8000.0,
        "Salary": 150000.0
      },
      "top_expenses": [
        {"category": "Health & Fitness", "amount": 15000.0},
        {"category": "Transport", "amount": 8000.0}
      ]
    }
  }
}
​```
```

**Правила:**

- В `categories` включить все категории с ненулевыми суммами (расходы отрицательные, доходы положительные).
- В `top_expenses` — топ-5 категорий расходов по абсолютной сумме.
- `savings_rate` — норма сбережений в процентах (для отслеживания динамики).

## Ограничения

- НЕ создавать, не обновлять, не удалять транзакции в ZenMoney. Только чтение.
- НЕ давать общие финансовые советы без привязки к конкретным данным.
- НЕ конвертировать валюты — анализировать каждую валюту отдельно.
- НЕ показывать секции отчёта, для которых нет данных.
- Если ZenMoney недоступен или возвращает ошибку — сообщить пользователю и прекратить выполнение.
